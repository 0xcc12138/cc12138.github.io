为什么计算机的操作数据的单位是二进制？

和芯片的组成有关，芯片是由电子器件二极管组成，正向加电则通，反向加电则不通，所以有两种状态。

硬件运行机制

机器码
类似111111000010101010B，可以用来控制硬件的二进制数据，叫做机器码。

助记符
二进制值难记，每种功能的二进制控制码取一个容易记住的名字，叫做助记符，也称之为指令例如，

00B-add
01B-sub
10B - xor

汇编
助记符硬件不能识别，需要将其转换成对应的机器码,这个过程叫做汇编。

一个典型的硬件系统组成
一个系统不可能由一个硬件单独完成，所以划分出多个硬件模块然后由一个硬件模块居中调度，称作cpu (centeral processing unit)。

I/O桥模块（Input/Output Module）是计算机系统与外部设备之间的连接桥梁，它扮演着非常重要的角色。I/O桥模块的主要功能包括处理器通信、设备通信、控制和定时、数据缓冲以及错误检测。它能够实现处理器与I/O模块之间的数据传输，接受和解码处理器发送的命令，报告当前状态，并具有识别自身唯一地址的能力。此外，I/O桥模块还能够管理计算机内部资源与连接的外部设备之间的数据流，以及检测和报告机械或数据错误。没有I/O桥模块，处理器与外围设备之间的数据交换将无法实现，也无法将数据解释为可供通信和消费的信息。

标志寄存器

进位标志（Carry Flag）：

进位标志寄存器CF，在Dosbox看最后一个，NC代表没进位，CY代表进位（应该是看有无溢出）
例如:
mov ax,ff
add ax,1
这样CF寄存器就没有被改变

mov al,ff
add al,1
这样寄存器是会被改变的

零标志寄存器ZF

若运算结果为0，则ZF=1，否则ZF=0
用途：用于表示两个无符号数的高低
举例：

3AH + 7CH =B6H 结果不是0：ZF=0

84H+7CH=(1)00H ,结果是0，ZF=1

NZ是ZF=0 ZR是ZF=1

溢出标志寄存器OF （Overflow Flag）

有符号数看OF，无符号数看CF

符号标志SF

运算结果最高位为1，SF为1，否则为0

有符号数据用最高有效位表示数据的符号，最高有效位是符号标志的状态

奇偶标志位PF

当运算结果中，指低八位中1的个数为偶数时置为1，否则为0

4.寄存器间接寻址

操作数存在内存中，段内偏移存储在寄存器中的寻址方式称为寄存器间接寻址

间接寻址寄存器有BX BP SI DI

​ MOV AX,[SI]1000

6.基质变址寻址

操作数值在内存中，段内偏移由[寄存器+寄存器]计算得来的寻址方式称作基质变址寻址

可以用作基地址的寄存器有BX,BP

BX默认DS段，BP默认SS段

可用变址的寄存器有SI,DI

MOV AX,[BP+BX]

MOV AX,[SI+DI]

标志寄存器传送指令

标志寄存器传送指令用来传送标志寄存器FLAGS的内容，方便进行对各个标志位的直接操作。

-16位传送:PUSHF和POPF

LAHF：

功能: 取FLAG标志寄存器低8位至AH寄存器.(AH)<–(FLAG)7~0

说明: 该指令不影响FLAG的原来内容,AH只是复制了原FLAG的低8位内容.

SAHF：

功能: 将AH存至FLAG低8位

说明: 本指令将用AH的内容改写FLAG标志寄存器中的SF、ZF、AF、PF、和CF标志,从而改变原来的标志位.

PUSHF

POPF

作用：将存储器操作数的有效地址传送至指定的十六位寄存器中

4.位运算：

指令：

AND OR XOR NOT TEST

  1. AND (按位与):

    • 功能: 将两个操作数的对应位进行与操作。
    • 示例: AND AX, BX 将寄存器 AX 和 BX 的值进行按位与操作，并将结果存储回寄存器 AX 中。

  2. OR (按位或):

    • 功能: 将两个操作数的对应位进行或操作。
    • 示例: OR CX, DX 将寄存器 CX 和 DX 的值进行按位或操作，并将结果存储回寄存器 CX 中。

    （只要有1就是1）

  3. XOR (按位异或):

    • 功能: 将两个操作数的对应位进行异或操作。
    • 示例: XOR SI, DI 将寄存器 SI 和 DI 的值进行按位异或操作，并将结果存储回寄存器 SI 中。

  4. NOT (按位取反):

    • 功能: 对操作数的每一位取反。
    • 示例: NOT AX 对寄存器 AX 中的每一位取反。

  5. TEST (按位测试):

    • 功能: 对两个操作数进行按位与操作，但不保存结果，仅影响标志位。
    • 示例: TEST CX, BX 对寄存器 CX 和 BX 的值进行按位与操作，但不保存结果，只影响标志寄存器。

1)如果要将目的操作数中某些位清0，用AND，称之为屏蔽

2)如果要将目的操作数中某些位置1，用OR

3)用来测试目的操作数中某一位或某几位是否为0或1，而目的操作数不变，TEST，

4)TEST与CMP的区别，前者是测试一位或几位，后者测试整个字节/字/双字是否相等

5)操作数自身相或、相与结果不变。

1.什么叫 ModR/M 字节 ？

ModR/M 字节是指令中的一个字节，用于指示指令的操作数。它通常由三个字段组成：Mod、Reg 和 R/M。

  1. Mod 字段（2 bits）：用于指示操作数的寻址方式（Addressing Mode）。
    • Mod=00：仅使用寄存器作为操作数（无需额外的位来指示寄存器）。
    • Mod=01：操作数为寄存器加一个字节的偏移量。
    • Mod=10：操作数为寄存器加一个字或双字的偏移量。
    • Mod=11：仅使用寄存器作为操作数（无需额外的位来指示寄存器）。
  2. Reg 字段（3 bits）：用于指示操作数的寄存器。
    • 这个字段用于指示源操作数或目的操作数中的寄存器。
  3. R/M 字段（3 bits）：用于指示另一个操作数。
    • 这个字段用于指示源操作数或目的操作数中的寄存器或内存地址。

2.条件跳转

依据标志位判断

  1. **JE (Jump if Equal)**：如果两个操作数相等，则跳转到目标地址。

  2. **JNE (Jump if Not Equal)**：如果两个操作数不相等，则跳转到目标地址。

  3. **JZ (Jump if Zero)**：如果结果为零，则跳转到目标地址。

  4. **JNZ (Jump if Not Zero)**：如果结果不为零，则跳转到目标地址。

  5. **JS (Jump if Sign)**：如果结果为负，则跳转到目标地址。

  6. **JNS (Jump if Not Sign)**：如果结果不为负，则跳转到目标地址。

  7. **JP (Jump if Parity)**：如果结果为偶数个1，则跳转到目标地址。

  8. **JNP (Jump if Not Parity)**：如果结果不为偶数个1，则跳转到目标地址。

  9. **JO (Jump if Overflow)**：如果发生溢出，则跳转到目标地址。

  10. **JNO (Jump if No Overflow)**：如果没有发生溢出，则跳转到目标地址。

  11. **JB (Jump if Below)**：如果无符号数下溢，则跳转到目标地址。

  12. **JAE (Jump if Above or Equal)**：如果无符号数不小于，则跳转到目标地址。

  13. **JBE (Jump if Below or Equal)**：如果无符号数小于或等于，则跳转到目标地址。

  14. **JA (Jump if Above)**：如果无符号数大于，则跳转到目标地址。

  15. **JL (Jump if Less)**：如果有符号数小于，则跳转到目标地址。

  16. **JGE (Jump if Greater or Equal)**：如果有符号数大于或等于，则跳转到目标地址。

  17. **JLE (Jump if Less or Equal)**：如果有符号数小于或等于，则跳转到目标地址。

  18. **JG (Jump if Greater)**：如果有符号数大于，则跳转到目标地址。

  19. **ZF (Zero Flag)**：如果结果为零，则ZF被设置为1；否则为0。

  20. **SF (Sign Flag)**：如果结果为负，则SF被设置为1；否则为0。

  21. **PF (Parity Flag)**：如果结果具有偶数个位，则PF被设置为1；否则为0。

  22. **OF (Overflow Flag)**：如果发生溢出，则OF被设置为1；否则为0。

  23. **CF (Carry Flag)**：如果发生进位或借位，则CF被设置为1；否则为0。

主要记住
JZ/JE (zero,equal) ZF=1 相等/等于零

JNZ/JNE （no zero,no equal） ZF=0 不相等，不等于零

JCXZ CX is zero cx=0

无符号判断

JB below 小于

JA above 大于

JBE below or equal 小于等于

JAE above or equal 大于等于

有符号判断

JL less 小于

JG greater 大于

JLE less or equal 小于等于

JGE greater or equal 大于等于

调用约定：

C 调用约定（C Calling Convention）：在C语言中广泛使用的一种约定。根据C调用约定，函数的参数是从右往左入栈的，调用者负责清理堆栈上的参数，函数返回值一般存储在寄存器中

Stdcall 调用约定：主要用于Windows API函数。参数从右往左入栈，被调用的函数负责清理堆栈上的参数。
